Aim:
To detect if a singly linked list contains a cycle (loop) and, if present, locate the start of the cycle and remove it to restore the list back to a linear acyclic state.

Definition:
Floyd's Cycle Finding algorithm (Tortoise and Hare) is a pointer-based algorithm that determines whether a linked list contains a cycle. If a cycle exists, the algorithm mathematically derives the cycle entry point and safely rewires the boundary node next pointer to NULL.

Theory:
A cycle occurs when a node's next pointer points back to one of the previous nodes, creating an infinite loop during traversal.
1. **Cycle Detection:** We use two pointers, `slow` and `fast`. The `slow` pointer moves 1 step at a time, while the `fast` pointer moves 2 steps. If a cycle is present, they must meet at some node. If no cycle is present, `fast` reaches the NULL tail.
2. **Cycle Start Detection:** Let $D$ be the distance from the head to the cycle start, $P$ be the distance from the cycle start to the meeting point, and $C$ be the cycle length. The distance traversed by `slow` is $D + P$. The distance traversed by `fast` is $D + P + k \cdot C$. Since `fast` is twice as fast:
   $2(D + P) = D + P + k \cdot C \implies D + P = k \cdot C \implies D = k \cdot C - P$
   This equation proves that if we reset `slow` to the head and keep `fast` at the meeting point, then move both at the same speed (1 step at a time), they will meet exactly at the cycle start node!
3. **Cycle Removal:** Keep track of the cycle tail node (the node pointing to the start node) and set its next pointer to NULL.

Working Principle:
1. Initialize `slow` and `fast` pointers at the head of the list.
2. Loop while `fast` and `fast->next` are not NULL:
   - Move `slow` 1 step: `slow = slow->next`.
   - Move `fast` 2 steps: `fast = fast->next->next`.
   - If `slow == fast`, a cycle exists. Break loop.
3. If the loop terminates with no meeting point, return (no cycle).
4. Reset `slow` to head. Keep `fast` at the meeting point.
5. If `slow == fast` (cycle starts at head), traverse `fast` until `fast->next == slow` to find the tail.
6. Else, move both 1 step at a time until `slow->next == fast->next`. The node `slow->next` is the cycle start.
7. Break the loop by setting `fast->next = NULL` (or `curr->next = NULL` where `curr` is the cycle tail).

Advantages:
- Constant Space: Resolves cycle detection and removal in O(1) auxiliary space.
- Linear Time: Operates in O(N) time.

Disadvantages:
- Pointer Sensitivity: Minor pointer assignment mistakes can corrupt list references.
- Destructive: Rewires links, altering original memory structure.

Applications:
- Memory management: Detecting circular dependencies in heap allocations.
- Graph theory: Cycle detection in directed graphs.
- Garbage collection: Finding reference cycles.

Real Life Applications:
- Deadlock detection in distributed operating systems.
- Resource allocation checks in database engines.

Time Complexity:
O(N) where N is the number of nodes in the linked list.

Space Complexity:
O(1) auxiliary space (no additional storage structures needed).

Stable/Not Stable:
Not Applicable.

In-place/Not In-place:
In-place (mutates next pointers directly).

Prerequisites:
- Singly Linked list traversal.
- Floyd's tortoise and hare logic.

Common Mistakes:
- Forgetting to check if `fast` or `fast->next` is NULL before advancing (causes NullPointerException).
- Missing the edge case where the cycle starts at the head node.
- Infinite loops if pointers are not advanced properly.

When to Use:
- When checking linked list structures for loops before processing.
- When cleaning up circular configurations that could cause infinite loop hangs.

When Not to Use:
- When using system architectures that manage circular dependencies automatically.
- When lists are guaranteed to be acyclic.

Aim

The aim of the Next Greater Element algorithm is to find, for each element in an array, the first element on its right that is strictly greater than it. If no greater element exists to the right, the answer for that position is -1. The algorithm uses a monotonic stack to solve this problem efficiently in linear time.

Definition

For an element nums[i], its next greater element is the first value nums[j] such that j > i and nums[j] > nums[i]. The word "next" means the closest qualifying element when scanning from left to right after index i.

Theory

A direct solution checks every element to the right of each position until a greater value is found. This works, but it can take O(n^2) time for large arrays.

The optimized approach keeps a stack of useful candidates. While scanning from right to left, the stack stores elements that may become the next greater element for positions further to the left. Any element that is less than or equal to the current element can be removed because it cannot be the next greater element for the current position or for any earlier position that sees the current element first.

This creates a monotonic decreasing stack. The top of the stack, after removing smaller or equal elements, is the nearest greater value to the right.

Working Principle

The algorithm works by processing elements from right to left.

For each element:

1. Remove elements from the stack while they are less than or equal to the current element.
2. If the stack is empty, there is no greater element to the right, so store -1.
3. If the stack is not empty, the top of the stack is the next greater element.
4. Push the current element onto the stack because it may help elements on the left.

Algorithm Steps

1. Create an answer array of the same size as the input array.
2. Create an empty stack.
3. Traverse the array from the last index to the first index.
4. While the stack is not empty and the top value is less than or equal to the current value, pop the stack.
5. If the stack is empty, set the answer at the current index to -1.
6. Otherwise, set the answer at the current index to the stack top.
7. Push the current value onto the stack.
8. Return the answer array.

Example

Input:

nums = [2, 1, 3, 2, 5]

Processing from right to left:

- 5 has no greater element to its right, so answer is -1.
- 2 sees 5 as the next greater element.
- 3 removes 2, then sees 5 as the next greater element.
- 1 sees 3 as the next greater element.
- 2 removes 1, then sees 3 as the next greater element.

Output:

[3, 3, 5, 5, -1]

Advantages

- Runs in O(n) time.
- Avoids repeated scanning of the same elements.
- Works well for large arrays.
- Handles duplicate values correctly when the comparison is strictly greater.
- Forms the base technique for many stack-based array problems.

Disadvantages

- Requires extra space for the stack.
- The right-to-left traversal may be less intuitive for beginners.
- The algorithm must carefully remove equal values because equal values are not greater.

Applications

- Stock price analysis.
- Temperature prediction problems.
- Signal and sensor threshold detection.
- Finding the next stronger competitor or larger measurement.
- Circular array variants of next greater element.
- Query preprocessing where many indices ask for their next greater value.

Time Complexity

O(n)

Each element is pushed onto the stack once and popped from the stack at most once.

Space Complexity

O(n)

The stack can store up to n elements in the worst case, and the output array also stores n results.

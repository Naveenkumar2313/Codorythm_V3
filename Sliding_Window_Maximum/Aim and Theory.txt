Aim

The aim of the Sliding Window Maximum algorithm is to find the maximum value in each contiguous window of size k as the window slides through an array. Given an array of integers and a window size k, the task is to return an array containing the maximum value for every possible window position.

Definition

For an array nums and window size k, a window is a contiguous subarray of length k. The sliding window maximum returns an array result where result[i] is the maximum value in the window starting at index i, i.e., the maximum of nums[i], nums[i+1], ..., nums[i+k-1].

Theory

A brute-force approach checks each window independently, finding the maximum in O(k) time for each of the O(n) windows. This works, but it takes O(n * k) time for large arrays.

The optimized approach uses a monotonic deque to maintain indices of useful elements. The deque stores indices in decreasing order of their corresponding values. This ensures that the front of the deque always points to the index of the maximum value in the current window.

Working Principle

The algorithm works by processing each element and maintaining a deque of indices.

For each element at index i:

1. Remove indices from the front that are outside the current window (indices < i - k + 1).
2. Remove indices from the back whose values are less than nums[i], as they cannot be the maximum for any future window.
3. Add the current index i to the back of the deque.
4. Once the window is formed (i >= k - 1), add nums[window[0]] to the result.

Algorithm Steps

1. Create an empty deque to store indices.
2. Initialize an empty result array.
3. For each index i from 0 to n-1:
   a. While the deque is not empty and the front index is less than i - k + 1, remove from front.
   b. While the deque is not empty and nums[back index] is less than nums[i], remove from back.
   c. Add index i to the back of the deque.
   d. If i is at least k-1, add nums[front index] to the result.
4. Return the result array.

Example

Input:

nums = [1, 3, -1, -3, 5, 3, 6, 7]
k = 3

Processing:

- Index 0 (value 1): Deque is empty, add 0. Window not formed yet.
- Index 1 (value 3): Remove 0 (value 1 < 3), add 1. Window not formed yet.
- Index 2 (value -1): Add 2. Window [1, 3, -1] formed, max = nums[1] = 3.
- Index 3 (value -3): Add 3. Window [3, -1, -3] formed, max = nums[1] = 3.
- Index 4 (value 5): Remove 1, 2, 3 (all values < 5), add 4. Window [-1, -3, 5] formed, max = 5.
- Index 5 (value 3): Add 5. Window [-3, 5, 3] formed, max = 5.
- Index 6 (value 6): Remove 4, 5 (values < 6), add 6. Window [5, 3, 6] formed, max = 6.
- Index 7 (value 7): Remove 6 (value 6 < 7), add 7. Window [3, 6, 7] formed, max = 7.

Output:

[3, 3, 5, 5, 6, 7]

Advantages

- Runs in O(n) time.
- Each element is added and removed from the deque at most once.
- Uses O(k) extra space.
- Avoids the need to check each window independently.
- The deque provides O(1) access to the current maximum.

Disadvantages

- Requires understanding of deque operations.
- The index management can be tricky to implement correctly.
- Not as intuitive as the brute-force approach for beginners.

Applications

- Real-time data analysis for moving maximum values.
- Stock price monitoring with rolling windows.
- Signal processing for peak detection.
- Network traffic analysis for maximum bandwidth.
- Sensor data smoothing and outlier detection.
- Image processing for maximum filter operations.

Time Complexity

O(n)

Each element is pushed onto the deque once and popped at most once, so the total work is linear.

Space Complexity

O(k)

The deque can store at most k indices at any time, one for each element in the current window.

Aim:
To design and implement an algorithm to clone (deep copy) a linked list where each node has an additional "random" pointer that can point to any node in the list or null.

Definition:
Cloning a list with random pointers is a classic graph-copying problem. It requires replicating the structural layout of a list where nodes contain arbitrary cross-references, ensuring that no cloned node references any node in the original list.

Theory:
Deep cloning a singly linked list is straightforward: traverse the original list and copy next references. However, copies of random references are difficult because a random pointer might point to a node that has not yet been cloned or traversed, or it might create reference cycles.
There are two primary paradigms:
1. Hash Map Mapping: Use a hash table to store `map[original_node] = cloned_node` mappings, allowing random references to be resolved in a second pass.
2. In-Place Interleaving: Avoid extra memory by weaving cloned nodes directly into the original list between the original node and its next node. This temporary interleaving naturally stores the mapping of original to clone, enabling O(1) auxiliary space execution.

Working Principle:
### Method (Optimized In-Place):
1. **Interleave Copies:** Traverse the original list. For each node `curr`, create a clone `copy`. Insert `copy` directly between `curr` and `curr.next`.
   `Original(1) -> Clone(1) -> Original(2) -> Clone(2) -> null`
2. **Copy Random Pointers:** Traverse the interleaved list. For each original node `curr`, link its copy's random pointer:
   `curr->next->random = curr->random ? curr->random->next : NULL`
3. **Unweave Lists:** Separate the original and copy nodes into two independent lists, restoring the original list pointers and extracting the clone list.

Advantages:
- Space Efficient: The in-place interleaving method runs with O(1) auxiliary space.
- Structural Integrity: Safely replicates complex cyclic graph references without infinite loops.

Disadvantages:
- Multiple Passes: Requires at least three passes over the list in the space-optimized method.
- Destructive State: Temporarily breaks and mutates the original list structure during interleaving, making it unsafe if accessed concurrently.

Applications:
- Graph cloning: Copying general directed graphs with cross-references.
- Object serialization: Cloning complex configurations or memory layouts containing arbitrary pointers.
- Game state saving: Saving game stages where entities have complex reference layouts.

Real Life Applications:
- Copying memory buffers containing arbitrary pointers (deep cloning heaps).
- Cloning DOM structures with arbitrary dynamic attributes and link references.

Time Complexity:
O(N) where N is the number of nodes in the linked list (we traverse the list 2-3 times).

Space Complexity:
- O(N) for the HashMap method.
- O(1) auxiliary space for the In-Place interleaving method (excluding memory for the new nodes).

Stable/Not Stable:
Stable. Cloned nodes maintain their exact original structural sequence.

In-place/Not In-place:
In-place (for interleaving) or Not In-place (using HashMap). The output is a separate deep copy.

Prerequisites:
- Singly Linked list traversal and pointer swapping.
- Hash Maps (for map-based solution).

Common Mistakes:
- Forgetting to check if `curr->random` is NULL before accessing `curr->random->next`.
- Incorrectly unweaving the list, causing original pointers to remain linked to cloned nodes.
- Memory leaks from not cleaning up temporary node frames.

When to Use:
- When a complete deep copy of a list with complex cross-references is required.
- When memory is restricted (use the O(1) auxiliary space method).

When Not to Use:
- When only a shallow copy is required (sharing node references is sufficient).
- When list elements have simple sequential next links with no random pointers.
